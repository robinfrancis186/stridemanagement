## Scope

This pass focused on the live Supabase cutover regression, core authenticated route smoke tests, and the AI actions that were already failing in production.

## Root Cause Fixed

- Production was allowed to silently fall back to the seeded local demo backend.
- The client and API config used nullable-coalescing on trimmed env vars, which could ignore a valid fallback key when an empty publishable key was present.
- Resulting symptoms:
- Production showed seeded demo data and the demo admin identity.
- AI duplicate detection failed with `Authentication is required.`

## Code Changes

- Added shared runtime config helpers in `src/lib/runtime-config.ts`.
- Updated `src/lib/supabase.ts` to resolve the first non-empty Supabase URL/key and to make local backend mode explicit instead of silent.
- Updated `src/lib/firebase.ts` to use the explicit local-backend flag.
- Updated `api/_supabase.ts` so server routes resolve the same production Supabase values as the client.
